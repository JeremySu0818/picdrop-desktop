# PicDrop Tauri App

PicDrop is packaged as a Tauri 2 app. The GitHub Pages frontend is pulled from
the upstream repo, patched for Tauri, and generated into `src` before builds.

## Commands

- `npm run sync:page` manually refreshes the generated `src` copy.
- `npm run build` syncs the GitHub Pages frontend, patches it, and builds web
  assets.
- `npm run desktop:build` syncs, builds the web assets, then builds the Windows
  desktop app and installers. Release assets are copied to `output/` with
  normalized names.
- `npm run desktop:dev` syncs and runs the desktop app in development mode.
- `npm run android:init` generates or refreshes the Android project.
- `npm run android:build` syncs, builds web assets, and builds a signed
  installable release APK for mainstream ABIs only: `aarch64` and `x86_64`.
- `npm run android:build:phone` builds a signed release APK for Android phones
  only: `aarch64`. Signed APKs are copied to `output/` with normalized names.
- `npm run ios:init` and `npm run ios:build` are for macOS with Xcode.

## Notes

- The original server API URL is unchanged:
  `https://picdrop-server.jeremytw.qzz.io`.
- `src/` and `index.html` are generated by `npm run sync:page` and are ignored
  by git.
- `npm run android:build` creates a local release keystore under
  `src-tauri/gen/android/signing/` when one does not already exist. Keep that
  folder private; it is ignored by git.
- `solid-glass`, `fflate`, and the Outfit font are bundled locally instead of
  loading from `esm.sh` or Google Fonts at runtime.
- iOS builds require macOS, Xcode, and the Tauri iOS CLI support available on
  that platform.
